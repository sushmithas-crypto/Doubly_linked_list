1. Insert an Element at a Given Position

Steps:
1. Traverse to the node at position 1 (current node with data 20)
2. Create a new node with data 15
3. Set new node's prev pointer to point to node at position 0
4. Set new node's next pointer to point to current node at position 1
5. Update the next pointer of node at position 0 to point to the new node
6. Update the prev pointer of node at position 1 to point to the new node
Time Complexity: O(n) - where n is the position
Space Complexity: O(1)

2. Delete an Element at a Given Position

Steps:
1. Traverse to the node at position 1 (node with data 20)
2. Store references to the previous node (pos 0) and next node (pos 2)
3. Update the next pointer of the previous node to point to the next node
4. Update the prev pointer of the next node to point to the previous node
5. Free the memory of the deleted node (shown faded in diagram)
Time Complexity: O(n) - where n is the position
Space Complexity: O(1)

3. Print the Size of the List

Algorithm:
1. Initialize a counter variable count = 0
2. Start from the head node
3. Traverse through each node until reaching NULL
4. Increment the counter for each node visited
5. Return the final count value
Time Complexity: O(n) - must traverse all n nodes
Space Complexity: O(1) - only uses a counter variable
Note: Some implementations maintain a size variable that's updated during insertion/deletion for O(1) size retrieval

4. Print the List (Forward and Backward Traversal)

Forward Traversal (Head ‚Üí Tail):
1. Start at the head node
2. Print the current node's data
3. Move to the next node using the next pointer
4. Repeat until reaching NULL

Backward Traversal (Tail ‚Üí Head):
1. Start at the tail node
2. Print the current node's data
3. Move to the previous node using the prev pointer
4. Repeat until reaching NULL
Time Complexity: O(n) - visits each node once
Space Complexity: O(1) - only uses a traversal pointer
Advantage: Unlike singly linked lists, doubly linked lists can be traversed in both directions efficiently
